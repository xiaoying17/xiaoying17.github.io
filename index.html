Windows users are not happy #629

发表于 2018-05-17

在windows环境下用node-gyp作为一个依赖来初始化项目永远是一个痛点。本次利用vue框架，其中用到sqlite3依赖包，一直报错，gyp err… can’t find python等。根据出错提示，明白需要在win10环境下安装python,并设置其环境变量，还需要安装Visual C++的依赖环境。网上众说纷纭，总结教训，及时查看文档，github上有关于安装node-gyp的详细说明。

参考https://github.com/nodejs/node-gyp,在windows系统上安装node-gyp有以下两种方法。
其一是通过npm自己提供的依赖包来配置系统所需的工具和配置，具体命令为npm install –global –production windows-build-tools,以管理员身份在命令行运行。体会到一点，由于这些依赖包也是与时俱进一直在变化的，不是说这个命令就能完全解决问题，存在版本的更迭和一系列发现issue和完善bug的过程。
其二是手动添加依赖环境。第一步，安装visual C++构建环境，笔者选择options1，安装默认的Visual C++ Build Tools(http://landinghub.visualstudio.com/visual-cpp-build-tools )。第二步，安装Python2.7版本同时设置系统环境变量。第三步，nom config set msvs_version 2015。另外，如果安装了多个Python版本，需指明一个版本。

其他收获: Windows系统上Node.js环境配置，以上过程即为设置的过程。而sqlite3包恰好可以用来检验这个环境是否配置正确。此外，还可以安装node-sass等来检验。当然，不需要时，再通过npm uninstall卸载。
