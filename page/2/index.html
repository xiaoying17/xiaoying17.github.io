Windows users are not happy #629

发表于 2018-05-17

在windows环境下用node-gyp作为一个依赖来初始化项目永远是一个痛点。本次利用vue框架，其中用到sqlite3依赖包，一直报错，gyp err… can’t find python等。根据出错提示，明白需要在win10环境下安装python,并设置其环境变量，还需要安装Visual C++的依赖环境。网上众说纷纭，总结教训，及时查看文档，github上有关于安装node-gyp的详细说明。

参考https://github.com/nodejs/node-gyp,在windows系统上安装node-gyp有以下两种方法。
其一是通过npm自己提供的依赖包来配置系统所需的工具和配置，具体命令为npm install –global –production windows-build-tools,以管理员身份在命令行运行。体会到一点，由于这些依赖包也是与时俱进一直在变化的，不是说这个命令就能完全解决问题，存在版本的更迭和一系列发现issue和完善bug的过程。
其二是手动添加依赖环境。第一步，安装visual C++构建环境，笔者选择options1，安装默认的Visual C++ Build Tools(http://landinghub.visualstudio.com/visual-cpp-build-tools )。第二步，安装Python2.7版本同时设置系统环境变量。第三步，nom config set msvs_version 2015。另外，如果安装了多个Python版本，需指明一个版本。

其他收获: Windows系统上Node.js环境配置，以上过程即为设置的过程。而sqlite3包恰好可以用来检验这个环境是否配置正确。此外，还可以安装node-sass等来检验。当然，不需要时，再通过npm uninstall卸载。

今天在codepen上看到一个推特点赞的动画，来学习一下。材料需要一张雪碧图，链接在这里。图有29帧。这个动画通过设置transition来改变background-position背景图像的起始位置来实现。

深克隆与浅克隆

发表于 2018-04-01

区别

深克隆与浅克隆针对的是js中的复杂数据类型(Object,Array等)。对象的实例是存储在堆内存中然后通过一个引用值去操作对象，因此有复制引用和复制实例两种情况，极为浅克隆与深克隆的区别。
浅克隆是复制引用，复制后的引用都是指向同一个对象的实例，彼此间的操作会互相影响。而深克隆是在堆中重新分配内存，把源对象实例的所有属性都进行新建复制，复制后的对象与原来的对象是完全隔离的。

http笔记

发表于 2018-03-09

MIME类型

服务器为所有HTTP对象数据附加一个这个类型，浏览器查看这个类型来看如何处理这个对象。html中meta里的信息，如Content-type: image/jpeg Content-length:12984。

URL

如http://www.yahoo.com/images/logo.gif，分为三部分 http协议+服务器的因特网地址+指定Web服务器上的某个资源。
在HTTP客户端向服务器发送报文之前，需要用网际协议(IP)地址和端口号在客户端与服务器间建立TCP/IP连接。

基本的浏览器连接处理

1 浏览器从URL中解析出服务器的主机名;
2 DNS将主机名换成IP地址;
3 将端口号解析出来;
4 浏览器建立一条与Web服务器的TCP连接;
5 浏览器向服务器发送请求报文;
6 服务器向浏览器回送响应报文;
7 关闭连接。

[源IP地址、源端口号、目的IP地址、目的端口号]定义了一条连接。IP分组包括IP分组首部，TCP段首部，TCP数据块。
HTTPS是在安全的传输层上发送的HTTP。HTTP安全层是通过SSL及现代替代协议TLS来实现的。如果URL的方案为https，客户端会打开一条到服务器端口443的连接，然后与服务器握手，以二进制格式与服务器交换一些SSL安全参数，附上加密的HTTP命令。

加密的HTTPS事务:

1 建立到服务器端口443的TCP连接
2 SSL安全参数握手
3 在SSL上发送HTTP请求/在TCP上发送已加密的请求
4 在SSL上发送HTTP响应/在TCP上发送已加密的响应
5 SSL关闭通知。
其中，SSL安全参数握手包括:
a 客户端发送可供选择的密码并请求证书
b 服务器发送选中的密码和证书
c 客户端发送保密信息； 客户端和服务器生成密钥
d 客户端和服务器互相告知，开始加密过程。
